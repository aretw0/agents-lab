# TASK-BUD-079 — Plan (2026-04-21)

Promover a calibração de qualidade (soft intent) para superfície **distribuível** da `pi-stack`, evitando dependência de ajustes locais em `.pi/monitors/*` (ignorado por padrão).

## Problema
- A calibração inicial foi feita em arquivos locais de monitor no workspace.
- Esses arquivos não são fonte de distribuição; logo, a melhoria não propaga de forma confiável para usuários.

## Estratégia incremental
1. **Contrato documental explícito**
   - declarar no guia de monitoria que `.pi/monitors/*` é override local (workspace), não baseline distribuível.
2. **Plano de promoção técnica**
   - centralizar defaults de calibração em extensão versionada (`monitor-provider-patch` / config compartilhada);
   - manter override local apenas como exceção opt-in.
3. **Evidência de saída**
   - inspeção de docs + estado `.project` com task em progresso;
   - próximo slice: implementação mínima em extensão + smoke/inspeção.
